对用户生活方式有改变的新产品，如何获得用户培养用户习惯

发表于 2017-03-02 | 分类于 产品 |

一、 确定一下分析的框架

  1. 题目是： （对用户生活方式有改变的）新产品，如何 获得用户 + 培养用户习惯，拆开就是：新产品如何获得用户＋培养用户习惯。所以我们不需要讨论什么样的产品才是对用户生活方式有改变的产品，或者要改变生活应该怎么样，重点应该放在“如何”上面
  2. “获得用户”不应该单指增加了一个注册用户，这主要是推广方面的事情，如果用户行为只到注册这一步就截止了，那么不应该算获得了这个用户（这个概念比较抽象，不严谨）。所以我觉得“获得用户”和“培养用户习惯”可以放在一起讨论

二、用户为什么要用它

培养用户习惯是需要一个过程的，在这个过程中要让用户愿意（经常）使用它。

那么用户为什么要用它呢？ 付出收益比

用户使用新产品需要付出成本，其中最主要的是学习成本、经济成本和时间成本（跟崔瑞说的一样），而收益主要也是经济收益和时间收益（还有精神上的收益，比如娱乐和社交）。

  • 有时候付出时间成本换取经济收益
  • 有时候付出经济成本换取时间收益
  • 有时候是在赌博，承受可能付出成本的风险，赌自己能获得收益

一个改变（或者说改善）生活的产品要成立，那么用户的收益一定要大于付出（至少整体来说需要）。而在用户养成习惯之前（也就是培养期），用户需要额外付出 学习成本 ，因此用户的收益同等也要增大（或者时间和经济的付出相对减少），所以在这个期间需要进行大量的补贴，而经济补贴（烧钱）则是一种最直接的方式。还有就是增大精神收益，但是这个难，对于非社交或者娱乐性产品来说，有时候需要强有力的运营活动。

说了这么多，好像我只说了一点，要烧钱（这特么不是废话么= =）

如果只有这样的话你们肯定会打我，为了不被你们打，我还有一个想法

三、不要一次培养用户太多的习惯

这个意思就是说，作为一款新产品，不要有太多需要用户改变现有习惯才能使用的功能。

比如说现在是大哥大时代，你给用户一个 IPHONE 7，一下子让他衣食住行娱乐全部在线上解决，那用户肯定学不来，除非你每天给他很多很多很多钱让他用。

我们在一个项目最开始启动的时候，通用的做法是开发一个MVP（最小可行性方案），当用户接受了这个MVP后，再一步一步添加新功能。这里不仅有开发量的考虑，也有用户接受度的考虑，任何事情都要一步一步的来，培养用户习惯也是。

其实这个说白了就是降低用户学习成本，只保留最根本不可避免的部分。

所以我觉得用户是否愿意使用这个产品的根本原因是付出收益比，只是在培养用户习惯的阶段需要想办法抵消掉用户需要额外付出的学习成本。

工具类产品如何保持用户黏性

发表于 2017-03-01 | 分类于 产品 |

0. 什么是工具类产品

有些人对工具类产品的定义并不统一，其实本身也很难统一，每个人有每个人的理解。但是在很多应用商店里，是没有单纯的 工具 这个分类的，只有 系统工具 这个分类，因此我觉得没必要纠结工具类产品的定义。

1. 工具产品保持用户粘性的手段有哪些？

我跟其他做产品的朋友聊了很多，有说社交的；有说满足个性化需求的；有说体验和功能强大的···

我的想法主要有下面几点：

  • 体验和功能是基础，但是也有被超越的一天，工具类 APP 要保持黏性必须要有不可替代性
  • 工具虽然不能只是工具，但是不建议强行展开至与本身不切合的领域，比如支付宝做社交
  • 跟我对 SaaS 类产品的想法类似，工具产生数据，数据产生价值。而且用户数据是会累加的，其价值很难因为其他竞品的成长而被超越，这些数据才是不可替代和可以黏住（或者叫绑住）用户的

PS. 并不是说体验和功能不重要，这些是基础，只是不具有不可替代性

发表于 2016-09-05 | 分类于 产品 |

什么是 Python？

Python 是一门优雅、明确、简单的编程语言，拥有丰富和强大的库，适合编写脚本与进行数据分析

为什么我要学 Python？

我现在以及未来要做的都是产品，不是程序员，那么，为什么我要学 Python 呢？

一、提升工作效率

我的编程启蒙老师——浙江大学的翁恺曾经说过：“重复7次以上的动作就应该写代码（脚本）来做”

在这个时代，作为一个互联网人，几乎时刻都离不开计算机（包括移动设备）。我们会通过计算机处理很多数据和文件，当数据和文件量大到一定程度，或者重复次数很多的时候，通过代码（脚本）来实现我们的需求，在一定程度上能够解放我们的工作量，提升我们的工作效率。

大学就过去一年了，时间依旧过得那么快，还记得上大学之前就一直想象并期待着美好的大学生活，现在一年过去了，当初的未来变成了过去，还是应该回顾并记录下来的。好好总结一下那些好的与不好的，然后才能更好地规划接未来。

我觉得这些经历、成长和认识的朋友，才是人生中最重要的。
